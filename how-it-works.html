HOW IT WORKS

Understanding the conversion process

1

Upload Your Java Pack

Start by uploading your Java Edition texture pack (.zip format). Our system accepts packs of all versions and sizes.

2

Structure Analysis

Our converter analyzes the structure of your pack, identifying all texture files, models, and metadata.

3

Bedrock Conversion

Files are reorganized to match Bedrock's structure, paths are remapped, and textures are validated.

4

Manifest Creation

A proper manifest.json is generated with new UUIDs and includes your custom attribution.

5

Pack Assembly

The converted files are packaged together into a new .mcpack file ready for Bedrock Edition.

6

Download

Your freshly converted texture pack is ready to download and use in Minecraft Bedrock Edition!

Technical Conversion Details

File Structure Differences

Java Edition
  • assets/minecraft/textures/block/
  • assets/minecraft/textures/item/
  • assets/minecraft/textures/entity/
  • assets/minecraft/models/
  • assets/minecraft/blockstates/
  • pack.mcmeta
→
Bedrock Edition
  • textures/blocks/
  • textures/items/
  • textures/entity/
  • models/
  • blocks.json
  • manifest.json

What Gets Converted

  • ✓

    Textures

    All texture files (.png) are reorganized to their proper locations and paths while maintaining resolution.

  • ✓

    Pack Metadata

    Pack information from pack.mcmeta is transferred to the new manifest.json with proper formatting.

  • ✓

    Block Models

    Models are converted to Bedrock's geometry format where possible.

  • ~

    Animations

    Basic animations are supported, though complex animations may require manual adjustment.

  • ~

    Shaders

    Limited shader support due to fundamental differences between Java and Bedrock rendering.
